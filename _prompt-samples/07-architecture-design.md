# アーキテクチャ設計プロンプト

## 用途
- 特定のビジネス要件に対するAzureアーキテクチャ設計を作成
- Well-Architected Framework に基づく包括的な設計ガイド

## プロンプト

```
[ビジネスシナリオ]に対する Azure アーキテクチャを設計して、[試験名]試験対策として詳細に説明して。

# 設計要件
- 信頼性（Reliability）
- セキュリティ（Security）
- コスト最適化（Cost Optimization）
- 運用の優秀性（Operational Excellence）
- パフォーマンス効率（Performance Efficiency）

# 含めるべき要素
- アーキテクチャ図
- 各コンポーネントの選択理由
- データフロー設計
- セキュリティ実装
- スケーリング戦略
- 災害復旧計画
- 運用・監視戦略

# 出力形式
- 視覚的なアーキテクチャ図（ASCII または説明）
- 各レイヤーの詳細説明
- 実装手順
- 運用考慮事項
- コスト見積もりのガイダンス
```
